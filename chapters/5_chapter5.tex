\chapter{算法验证与性能评估} 

本章旨在对前文提出的非全维位置量测条件下的分布式状态观测器与鲁棒协同控制策略进行验证与性能评估。本章首先依托MATLAB数值仿真平台，针对无人机集群的系统可观测性、状态估计收敛性、机间避碰与通信连通性等核心理论指标展开系统性验证；随后，利用更贴近真实物理环境的Gazebo三维动力学仿真框架，为算法从理论迈向实际工程部署提供验证支撑。

\section{MATLAB仿真实验与结果分析}
\label{MATLAB仿真实验与结果分析章节}
为验证所设计编队控制协议的有效性，本节在 MATLAB 平台基于欧拉数值积分法构建了多无人机系统数值仿真环境。

\subsection{仿真参数设置与拓扑配置}
仿真场景设定为三维空间中由$N=4$架无人机组成的微型集群系统，其动力学行为由公式(\ref{一阶动力学})所描述。假设各无人机底层执行机构具备理想的宽频响应特性，能够无静差跟踪上层生成的期望速度指令$\mathbf{v}_i^*$。

首先，验证编队局部收敛的具体配置与章节\ref{异构传感器配置实例分析}一致。无人机集群的底层通信与传感拓扑为全连通图，共包含$M=6$条双向通信链路。为了满足第3章提出的系统弱可观测性要求，对4架无人机进行了异构传感器配置，即部分无人机仅能量测特定轴向的绝对位置，而另一部分仅具备相对测距能力。系统的期望编队构型被设定为一个边长为$\sqrt{2}$m的正四面体。编队$\mathbf{p}^*$在全局坐标系下的位置分别为$\mathbf{p}_1^*=[1;1;1]$，$\mathbf{p}_2^*=[2;1;2]$，$\mathbf{p}_3^*=[1;2;2]$，$\mathbf{p}_4^*=[2;2;1]$，单位均为m。测距传感器网络与通信网络相同，均建模为无向图，由边$\mathcal{E}=\{(i,j) \mid i,j\in \mathcal{I}_{4}\}$表征。另外，仿真设置中还包含了一些增益参数，详情见表\ref{无人机集群系统刚性编队仿真基础参数表}。根据距离约束可以计算出无人机初始位置的邻域参数$\rho_p=0.29$，基于此可以对无人机的初始位置$\mathbf{p}_i(0)$进行配置，初始位置估计$\hat{\mathbf{p}}_i(0)$根据初始位置随即给出。设定$k_\gamma=0.001$，可计算得参数$\rho_\gamma=0.003,\rho_\Gamma=0.013$。此时根据条件(\ref{控制反馈增益条件})和(\ref{观测反馈增益条件})可以计算出控制器和观测器增益的边界分别为1.2384和4.5522，从而设置反馈增益$k_c$和$k_o$。此外，为验证系统在初始偏差下的鲁棒收敛能力，所有无人机的初始实际位置与初始估计位置均在期望位置的基础上叠加了服从均匀分布的随机扰动。此外，为了模拟真实传感器测量的噪声特性，在测距信息中引入了均值为0、方差为0.03的高斯白噪声。

\section{Gazebo物理仿真平台验证}
尽管在\ref{MATLAB仿真实验与结果分析章节}节中，基于 MATLAB 的数值仿真已从理论层面验证了分布式状态观测器与编队控制协议的渐近收敛性，但理想的质点运动学或二阶积分器模型未能充分反映真实物理世界中的多重约束。在实际工程部署中，四旋翼无人机面临着复杂的空气动力学效应、执行机构的物理饱和限制以及传感器数据的高频噪声与通信延迟。

为进一步验证本文所提控制算法在贴近真实物理环境下的鲁棒性与工程可行性，本节在基于 ROS 与 Gazebo 的高保真三维物理仿真平台开展了综合验证。

\subsection{物理仿真环境与系统架构搭建}
为深入验证所提方法在复杂环境与物理约束条件下的适用性，本文将无人机轨迹跟踪与编队控制算法部署至机器人物理仿真平台 Gazebo 中进行测试。具体而言，本研究采用了由 Aspire 与 Open Robotics 联合开发、面向 MBZIRC 2024 海事大挑战赛的数字孪生系统。该系统已在 GitHub 平台开源，其相关参数与场景设置均严格遵循该赛事的官方标准。该系统高保真地模拟了GNSS拒止的海洋环境，具体场景如图\ref{mbzirc_coast_env}所示。在该仿真环境中，可实例化配备多种机载传感器的四旋翼无人机模型，如图\ref{Gazebo无人机模型}所示。各无人机平台支持挂载定制化的感知模块、通信模块、传感器模块以及执行器模块等。仿真环境高度还原了真实世界的物理特性，包括海洋风浪扰动、无人机的空气动力学效应、电池续航衰减以及电机动力学响应等。同时，还将通信链路的丢包率与传感器量测噪声等非理想特性纳入了考量。系统底层基于 ROS 2 架构实现节点间的数据交互。仿真时为每架无人机分配独立的 ROS 节点，以此精确模拟分布式控制框架（关于 ROS 系统的详细机制可参考其官方技术文档）。鉴于该环境对无人机集群间的信息交互带宽施加了严格限制，本研究对机间传递消息的数据结构与字节大小进行了针对性优化，以满足系统的通信约束。

\subsection{仿真场景与参数配置}
基于上述前提，本节仿真实验将重点评估在纯距离量测条件下的无人机系统位置可观测性，以及所提控制策略在非理想约束下的综合适用性。在具体实验配置中，将海洋仿真环境设定为二级海况（平均风速 $3.6\,\text{m/s}$），并在海岸区域初始化 4 架无人机。实验所采用的初始状态参数与前文数值仿真基本保持一致，具体配置详见表\ref{无人机集群系统刚性编队仿真基础参数表}。然而，考虑到 Gazebo 物理仿真引擎的底层解算与节点通信频率显著低于理想的数值仿真环境，为保证物理层面的动态稳定响应，本节将协同控制器与状态观测器的反馈增益参数针对性地调整为 $k_c=2$ 与 $k_o=5$。
需要特别说明的是，受限于 GNSS 拒止环境的设定，仿真中无人机在世界坐标系下的位置真值无法被机载传感器直接获取。然而，为便于对算法精度进行量化评估，本研究对数字孪生系统的底层源码进行了适配性修改，强制提取了物理引擎中的位置真值，并通过 ROS 消息网络进行了同步发布与记录。考虑到本文所提方法依赖于系统局部收敛的理论前提，加之在 Gazebo 平台中无法直接初始化悬停于空中的节点，本仿真设计了专门的起飞预定位阶段。先利用全局绝对坐标引导无人机起飞，待集群空间构型大致进入目标正四面体编队的局部收敛域后，随即切断全局坐标注入，全面切换至本文所提算法，以此验证编队在受限条件下的渐近收敛性与动态跟踪性能。此外，在异构传感器的搭载配置上同样遵循前文策略。指定无人机$4$为锚点节点，赋予其全维全局位置量测能力，其余节点则仅搭载测距传感器以获取与邻居节点间的相对距离，具体的异构量测矩阵配置详见表\ref{异构量测配置表}。

\subsection{物理仿真结果与性能分析}

\section{本章小结}
本章围绕前文提出的分布式状态观测器与编队控制协议，从数值理论验证到高保真物理仿真，开展了完备的性能评估与工程可行性论证。首先，在 MATLAB 平台搭建了基于二阶积分器动力学的多机仿真系统，验证了纯距离与受限位置量测配置下，观测器与控制器闭环系统的局部渐近收敛性，并展示了平移、旋转及伸缩等典型编队机动能力。随后，进一步向工程实际靠拢，在 Gazebo 与 ROS 构架的数字孪生海洋环境中进行了综合测试。物理仿真结果表明，在叠加非线性空气动力学、底层闭环响应延迟及传感器异构降级等复杂真实约束后，系统依然能够依靠定制化底层模式与顶层鲁棒控制律保持 $0.1\,\text{m}$ 内的协同控制精度，且避碰机制实现了零碰撞的硬边界防护。仿真结果不仅印证了算法的理论正确性，更为其向真实微型集群硬件平台的移植奠定了坚实基础。

		import numpy as np
		
		def incmatrix(genl1,genl2):
		m = len(genl1)
		n = len(genl2)
		M = None #to become the incidence matrix
		VT = np.zeros((n*m,1), int)  #dummy variable
		
		#compute the bitwise xor matrix
		M1 = bitxormatrix(genl1)
		M2 = np.triu(bitxormatrix(genl2),1)
		
		for i in range(m-1):
		for j in range(i+1, m):
		[r,c] = np.where(M2 == M1[i,j])
		for k in range(len(r)):
		VT[(i)*n + r[k]] = 1;
		VT[(i)*n + c[k]] = 1;
		VT[(j)*n + r[k]] = 1;
		VT[(j)*n + c[k]] = 1;
		
		if M is None:
		M = np.copy(VT)
		else:
		M = np.concatenate((M, VT), 1)
		
		VT = np.zeros((n*m,1), int)
		
		return M
